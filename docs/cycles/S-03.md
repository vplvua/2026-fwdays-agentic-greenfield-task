## Що спрацювало

- **Launch-and-look як окремий крок DoD виправдався**: баг зі шрифтом іконок
  не ловиться жодним рівнем тестів (Playwright бачить лігатурний текст
  `home_work` як валідний DOM) — його видно лише очима на скриншоті.
- Ревʼю (ADR-0010) вперше **APPROVE з першого проходу**; 1 medium-знахідка
  реальна (whitespace-імʼя обходило клієнтську валідацію і губило ввід
  діалога) — виправлено + e2e. Канон ізоляції (single-query `id AND
  user_id`, 404-парність) ревʼюер перевіряв прицільно, включно з
  таймінг-каналами і mass-assignment — відхилив 8 кандидатів як не-знахідки
  з обґрунтуванням.
- Патерн «канонічної двокористувацької сюїти ізоляції» в api-e2e
  (GET/PATCH/DELETE чужого = байт-у-байт 404 неіснуючого) готовий до
  копіювання в S-04+ (план §4 п.4).
- Спільні логін-хелпери api-e2e винесено в `auth-helpers.ts` заздалегідь —
  дубляж не виник, fallow-дедуплікація мовчала.
- Смок на реальній БД через curl + перевірка рядків у MySQL зайняли ~2 хв
  і підтвердили trim/ізоляцію/очищення незалежно від тестових моків.

## Тертя і знахідки

- (інструменти) **fallow: два false positive поспіль на одному гейті**:
  1) `unused-component-input` для `health()`, який читається в шаблоні через
  `@else if (health(); as h)` — сканер шаблонів не бачить читання всередині
  виразу `@else if`; 2) інлайн-suppression для цього правила **не
  реєструється** (одночасно «issue на рядку N» і «suppression на N−1
  застаріла») — обійдено реструктуризацією шаблону (вкладений `@if` замість
  `@else if`). Разом ~20 хв.
- (інструменти) **семантика fallow-гейта неочевидна**: «inherited» знахідки
  незмінених файлів виключаються з гейта, але щойно торкаєшся файла (навіть
  додаючи suppression-коментар!) — його старі знахідки стають блокуючими.
  Перша спроба suppression сама втягнула файл у гейт.
- (інструменти) складність шаблону контейнера: мінімальна машина станів
  loading/loaded/error у `houses-page` дає cyclomatic 5–7 і HIGH; частково
  вилікувано виносом `house-list` (правильно за ADR-0009), решта —
  документований suppression. Поріг не налаштовується per-kind.
- (тести) Playwright strict mode: заголовок confirm-діалога повторює назву
  будинку → `getByText` резолвиться у 2 елементи; асертити треба на
  `app-house-card`, не на голий текст.
- (документація vs практика) tasks.md з propose-фази вимагав «400-style
  errors with Ukrainian messages» в API — суперечить locale-free контракту
  S-02 D6 (коди в API, українська в SPA). Виправлено під час apply; правило
  варто зафіксувати в `openspec/config.yaml`, щоб не відтворювалося.

## Покращення — застосовано (≤3)

- CLAUDE.md → Quality gates (fallow): задокументовано семантику гейта
  (дотик до файла втягує його inherited-знахідки) і два обхідні шляхи:
  читання інпутів у `@else if (…; as …)` сканер не бачить (реструктурувати
  у вкладений `@if`), suppression для `unused-component-input` не
  реєструється — не витрачати на нього цикл.
- openspec/config.yaml → правило proposal: контракт помилок API —
  locale-free машинні коди (`{code, message}`), українська копія лише в
  SPA (S-02 D6); нові capability наслідують його, а не вигадують свій.
