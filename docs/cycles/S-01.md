## Що спрацювало

- Повний SDD-контур за одну сесію: proposal → design → spec-дельти → tasks →
  код → e2e → adversarial-ревʼю → прод → archive; `npx openspec list` порожній.
- Verify gate у pre-commit хуку реально ловив проблеми (fallow — незадіяні
  залежності; prettier — неформатований spec після archive).
- Adversarial-ревʼю (ADR-0010) окупилось одразу: 2 high-знахідки, обидві
  реальні й відтворювані — «одна команда старту» падала на чистому клоні
  (prisma generate відсутній поза Docker-збіркою), застарілий api-e2e скафолд.
  Maker ≠ checker працює.
- Живі перевірки поведінки (а не лише тести): цикл health up→503→recovery
  зловив невідновлюваний пул зʼєднань (деталі нижче) ще до деплою.
- Railway CLI виявився достатнім для всього, крім тумблера бекапів: проєкт,
  MySQL, volume, env-змінні, деплой, домен — без дашборда.

## Тертя і знахідки

- (інструменти) **MySQL 8 `caching_sha2_password` + mariadb-драйвер**: після
  рестарту БД пул не відновлювався ніколи (auth-кеш сервера очищається, драйвер
  без `allowPublicKeyRetrieval` не може пройти повну автентифікацію). Найбільший
  шматок дебагу сесії; зафіксовано в design.md (Risks) і в PrismaService.
- (інструменти) **Prisma 7 сюрпризи**: дефолтний ESM-вивід генератора ламає
  jest/webpack CJS (`moduleFormat = "cjs"`); згенерований webpack-ом
  `package.json` не бачить deep-import `@prisma/client/runtime/*` — у Docker
  runtime-стадії клієнт ставиться явно з пінованою версією; target `api:prune`
  зі скафолда зламаний (очікує `api/package.json`) — не використовується.
- (інструменти) **nxE2EPreset падає під ESM-лоадером Playwright**
  (TypeError у nx native module) — конфіг web-e2e переписано без пресета,
  дефолти інлайновано. Playwright-браузери не були встановлені (перший запуск).
- (інструменти) **PreToolUse-хук verify спрацьовує до виконання всієї
  команди**: `prettier --write && git commit` в одній Bash-команді не працює —
  фікс має бути окремою командою перед комітом. Коштувало одного зайвого
  прогону verify (~2 хв).
- (інструменти) fallow: блокування «unused dependencies» на межі кроків
  (залежності встановлено раніше за код) — істинно-позитивне, але змушує
  комітити залежності разом з першим використанням; хибно-позитивне «input
  `health` is read nowhere» для читання в inline-шаблоні через
  `@else if (health(); as h)` (audit не блокує, лише шумить).
- (процес) Ревʼюер отримав діапазон `1a78cd3..HEAD`, а HEAD за час ревʼю
  поїхав уперед (Railway-коміт) — ревʼюер коректно це зазначив, але діапазон
  варто фіксувати кінцевим SHA і не комітити під час ревʼю.
- (середовище) Порт 3000 на машині зайнятий стороннім проєктом
  (`~/Projects/cabinet`) — двічі спантеличував перевірки (Spring-подібні 401,
  `reuseExistingServer` Playwright вважав чужий сервер своїм).
- (документація vs практика) `openspec archive` пише spec у не-prettier
  форматі → verify блокує наступний коміт; кроку «відформатувати після
  archive» ніде не було зафіксовано.
