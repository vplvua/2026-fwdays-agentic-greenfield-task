# ADR-0005 — Середовища й деплой: local + prod, Docker-образ на Railway

- **Статус:** Accepted
- **Дата:** 2026-07-06
- **Замінює / Замінено:** —
- **Трасування:** TC-OPS-01 · BC-COST-01 · NFR-REL-01 · кореневі ADR-0008/0009

## Контекст і проблема

Основний проєкт має prod + staging + local (кореневий ADR-0009) і драбину
міграції хостингу (ADR-0008). Для POC треба мінімальну, але робочу топологію
середовищ і деплою, придатну для агентних циклів «зробив — задеплоїв — подивився».

## Драйвери рішення

- **BC-COST-01** — мінімум платних середовищ.
- Мета POC — бачити результат кожного циклу на живому URL.
- TC-OPS-01 — секрети в env, TLS від платформи.

## Розглянуті варіанти

- **A — local + prod (Railway):** локальна розробка (docker-compose: MySQL;
  застосунок — nx serve) + один прод-сервіс на Railway.
- **B — local + staging + prod** — як в основному проєкті; для POC зайве.

## Рішення

Обрано **A**:

- **local:** MySQL у docker-compose; API+SPA — через Nx; SMS — dev-фолбек
  (ADR-0004); тека вкладень — локальна.
- **prod (Railway):** один сервіс із Docker-образу (ADR-0002) + керована MySQL +
  Volume для вкладень (ADR-0003). Деплой — з GitHub-репозиторію нового проєкту
  (Railway build з Dockerfile). Health-ендпоінт (NFR-OBS-01) для рестартів.
- **Бекапи:** БД — засобами Railway (добові); volume — не бекапиться (прийнято,
  ADR-0003).
- Міграція на Hetzner/Coolify та staging — **поза скоупом POC** (без тригерів:
  POC завершується, а не еволюціонує).

## Наслідки

- **Позитивні:** один платний контур; кожен агентний цикл закінчується перевіркою
  на живому проді; мінімум CI/CD-налаштувань.
- **Ціна:** немає staging — експерименти йдуть або локально, або прямо на прод
  POC (прийнятно: реальних користувачів немає).
