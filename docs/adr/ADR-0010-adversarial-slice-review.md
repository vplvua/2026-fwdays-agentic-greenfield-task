# ADR-0010 — Adversarial-ревʼю зрізу: субагент з чистим контекстом на іншій моделі

- **Статус:** Accepted
- **Дата:** 2026-07-07
- **Замінює / Замінено:** —
- **Трасування:** BC-PRIN-01 · BC-COST-01 · принцип maker ≠ checker · ADR-0006 · ADR-0008

## Контекст і проблема

Механічні чекери (verify gate, fallow, Playwright) не ловлять логічних
помилок, відхилень від специфікації та «мовчазних» продуктових рішень у коді.
Потрібне код-ревʼю з трьома властивостями: **чистий контекст** (ревʼювер не
бачить міркувань автора), **інша модель** (різні сліпі зони) та **критичність
за замовчуванням** (порожнє ревʼю = провал ревʼю). Через ADR-0008 робочих PR
немає — PR-based ревʼю (CodeRabbit) під час розробки недоступне.

## Розглянуті варіанти

- **A — субагент `slice-reviewer`** (Claude Code agent, read-only, чистий
  контекст, модель відмінна від автора).
- **B — CodeRabbit на по-зрізових PR:** вимагає повернути гілки+PR — відкат
  ADR-0008.
- **C — CodeRabbit CLI на локальному diff:** робочий, але зовнішня залежність
  з auth і лімітами (BC-COST-01), вердикти повертаються в контекст автора.
- **D — ревʼю в сесії автора:** немає незалежності — автор ревʼюїть сам себе.

## Рішення

Обрано **A**:

- Агент `.claude/agents/slice-reviewer.md`: **модель Sonnet 5** (сесія-автор —
  Fable 5), read-only інструменти, свіжий контекст без історії діалогу.
- **Один прогін на зріз**: після verify + e2e, **перед archive**; вхід — ID
  зрізу та діапазон комітів.
- Промпт вимагає: спершу нормативний контекст (план зрізу, FR-коди PRD, ADR,
  конвенції), потім повний diff; мінімум 5 розглянутих кандидатів у проблеми;
  кожен — підтверджений як знахідка або відхилений з перевіреною причиною;
  структурований вивід (severity, file:line, сценарій відмови).
- **Блокуючість:** знахідки `critical`/`high` ⇒ BLOCK — виправити й повторити
  verify до archive; `medium`/`low` — на розсуд автора, диспозиції фіксуються
  в ретро зрізу (`/slice-retro`).
- CodeRabbit лишається зовнішнім чекером **фінального PR** здачі (конфіг
  `.coderabbit.yaml` читається з head-гілки). Варіант C (CodeRabbit CLI як
  другий ревʼювер) — кандидат на додавання через ретро, якщо знадобиться.

## Наслідки

- **Позитивні:** maker ≠ checker став явним кроком DoD з артефактом
  (вердикт + список відхилених кандидатів доводить, що ревʼювер дивився);
  різні моделі автора й ревʼювера; нуль зовнішніх залежностей.
- **Ціна:** додаткові токени на прогін (Sonnet 5 — компроміс вартості);
  ревʼювер без контексту діалогу може повторно піднімати вже свідомо
  прийняті рішення — ліки: фіксувати такі рішення в Р-/ADR, а не в діалозі.
