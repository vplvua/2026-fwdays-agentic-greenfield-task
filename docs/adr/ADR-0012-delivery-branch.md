# ADR-0012 — Гілка постачання `service-desk-mini`; `main` дзеркалить стартер курсу

- **Статус:** Accepted
- **Дата:** 2026-07-08
- **Замінює / Замінено:** уточнює [ADR-0008](./ADR-0008-trunk-based-no-working-prs.md) (сам флоу незмінний)
- **Трасування:** вимоги здачі README (форк → окрема гілка → PR) · BC-PRIN-01

## Контекст і проблема

ADR-0008 зафіксував trunk-based постачання «коміти в `main`». Але правила
здачі домашки (README) вимагають: проєкт лежить у форку **на окремій гілці**,
здача — PR, який CodeRabbit ревʼюить проти чистого `main` (стартер курсу).
S-01 було закладено в `main` — виявлено після завершення зрізу.

## Рішення

- **Трунк проєкту — гілка `service-desk-mini`.** Все з ADR-0008 діє без змін
  (зрізи — серіями `feat(S-NN)` комітів, без робочих гілок/PR), лише трунк
  тепер ця гілка, а не `main`.
- **`main` форку дзеркалить стартер upstream** (`75fe221`) і не отримує
  проєктних комітів — він база фінального PR.
- **Фінальна здача:** один PR `service-desk-mini` → `main` у форку
  (CodeRabbit ревʼюить у форку), посилання на PR — здача.
- Виправлення застосовано 2026-07-08: гілку створено на вершині історії
  (жоден коміт не втрачено), `main` force-push-нуто назад на `75fe221`.

## Наслідки

- **Позитивні:** відповідність правилам здачі; фінальний PR показує ввесь
  проєкт одним дифом проти стартера; історія комітів незаймана.
- **Ціна:** локальні клони мають працювати на `service-desk-mini`;
  `fallow audit` рахує diff від upstream поточної гілки (перевірено —
  працює коректно без змін конфігу).
