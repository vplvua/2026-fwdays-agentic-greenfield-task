# ADR-0009 — Архітектура фронтенду: signals + facades, container/presentational

- **Статус:** Accepted
- **Дата:** 2026-07-07
- **Замінює / Замінено:** —
- **Трасування:** TC-STACK-01 · TC-UI-01 · BC-PRIN-01 · NFR-COMPAT-01

## Контекст і проблема

Зрізи S-02…S-08 додаватимуть Angular-код різними агентними сесіями. Без
зафіксованих принципів кожен зріз прийматиме архітектурні мікрорішення
(розділення компонентів, стейт, структура тек) заново — результат буде
неконсистентним. Водночас застосунок малий (5 екранів, PRD §6), тож повна
архітектура великого проєкту була б надлишковою (BC-PRIN-01).

## Драйвери рішення

- Консистентність між агентними сесіями; прості патерни генеруються краще.
- Angular 21: скафолд уже zoneless (без zone.js), standalone, файли без
  суфіксів (`app.ts`).
- BC-PRIN-01 — мінімум шарів і церемонії.

## Розглянуті варіанти

- **A — signals + facades** (сервіс-фасад на фічу зі станом на signals),
  два шари компонентів: container / presentational.
- **B — NgRx / SignalStore:** DevTools і time-travel, але boilerplate;
  undo/redo не потрібен — головна перевага втрачається.
- **C — без правил** (кожен зріз вирішує сам): неконсистентність.

## Рішення

Обрано **A** (деталі й шаблони — скіл `web-conventions`):

- **Стейт:** один фасад на фічу — injectable-сервіс зі станом на `signal`/
  `computed`, публічні signals лише readonly; HTTP — окремий api-сервіс
  фічі. NgRx не використовується.
- **Компоненти:** два шари. **Container** (сторінка/секція) інʼєктить фасад,
  зʼєднує дані та події; **presentational** — лише `input()`/`output()`,
  без інʼєкцій, локальний UI-стан дозволено. Максимум два рівні контейнерів.
- **Реактивність:** zoneless; `ChangeDetectionStrategy.OnPush` всюди;
  нативний control flow (`@if`/`@for`); без прямих маніпуляцій DOM.
- **Структура:** `web/src/app/core/` (shell, guards, interceptors),
  `web/src/app/features/<feature>/` (containers / components / data),
  `web/src/app/shared/` — лише коли використовують ≥2 фічі. Окремі `libs/`
  не заводимо, поки не зʼявиться другий споживач (BC-PRIN-01).
- **Іменування:** стиль Angular 21 — файли kebab-case без суфіксів
  `.component`; роль видно з теки та класу (`HousesPage`, `HouseCard`);
  селектори з префіксом `app-`.
- Вибір UI-набору — окреме рішення S-01 (В-01), цим ADR не фіксується.

## Наслідки

- **Позитивні:** кожен зріз пише однаковий за формою код; presentational-
  компоненти тривіально тестуються; патерни прості для генерації.
- **Ціна:** без Redux DevTools/time-travel (прийнятно: стан малий);
  дисципліна readonly-signals тримається код-ревʼю та ретро (`/slice-retro`).
