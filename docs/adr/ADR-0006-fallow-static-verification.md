# ADR-0006 — Незалежна статична верифікація коду агентів: fallow

- **Статус:** Accepted
- **Дата:** 2026-07-07
- **Замінює / Замінено:** —
- **Трасування:** BC-PRIN-01 · BC-GOAL-01 · принцип maker ≠ checker ([agent-plan](../agent-plan.md))

## Контекст і проблема

Код у POC пишуть агенти, і агентний код має типові дефекти: мертві файли/експорти,
дублювання логіки, зайві залежності. CodeRabbit ревʼюїть логіку в PR, але не має
повного графа модулів. Потрібен незалежний від «мейкера» механічний чекер, який
дешево (секунди, без CI-інфраструктури) перевіряє цілісність кодової бази перед
кожним PR — у дусі maker ≠ checker.

## Драйвери рішення

- **BC-PRIN-01** — «не складніше за блокнот»: zero-config старт, без платних
  сервісів і CI-воріт.
- **BC-GOAL-01** — не витрачати цикли на інфраструктуру якості.
- Інтеграція з агентним циклом: структуровані MCP-інструменти + JSON-вивід.

## Розглянуті варіанти

- **A — fallow** (CLI + MCP): dead code, дублікати, complexity, `audit` з вердиктом
  pass/warn/fail по змінених файлах; статичний аналіз безкоштовний, open source.
- **B — knip + jscpd:** ті самі класи знахідок, але два інструменти, два конфіги,
  без єдиного audit-вердикту й без MCP.
- **C — нічого (тільки CodeRabbit):** нуль витрат, але без графового аналізу
  мертвого коду й дублів.

## Рішення

Обрано **A**:

- `fallow` — devDependency; конфіг — `.fallowrc.json` у корені (entry-точки
  `api/src/main.ts` і `web/src/main.ts`; тестова інфраструктура Jest виключена з
  аналізу; тулчейн-пакети — в `ignoreDependencies`).
- MCP-сервер `fallow-mcp` підключено в `.mcp.json` — агенти викликають
  `analyze`/`audit` як інструменти.
- Процесне правило: наприкінці кожного циклу перед PR запускається
  `npx fallow audit` — вердикт має бути «pass»; знахідки або виправляються, або
  свідомо глушаться конфігом із поясненням у PR.
- Платний runtime-шар fallow — **поза скоупом POC**.

Пілот на скафолді (Е-1): після разового конфігу — 0 хибних спрацювань на
продуктовому коді; впіймано 2 реальні дрібниці (невикористаний `@angular/forms`,
`axios` у prod-залежностях замість dev).

## Наслідки

- **Позитивні:** механічний чекер, незалежний від агента-мейкера; чистий сигнал
  за ~0.1 с; агенти отримують структуровані знахідки через MCP.
- **Ціна:** `.fallowrc.json` треба супроводжувати (нові рядково-підключувані
  файли типу Jest-сетапів можуть давати хибні «мертві файли»); інструмент молодий —
  якщо шум переважить користь, вимкнення = видалити devDependency і конфіг.
